最小靜態版部署Static Hello World Deployment

步驟一的目標是先證明：不靠 XPages，也能把純靜態前端包成 OSGi webapp，透過 UpdateSite NSF 部署到 Domino。 Step 1 proved that a pure static front end can be packaged as an OSGi webapp and deployed through an UpdateSite NSF without using XPages.

做了哪些事What changed

  • 建立最小 `hello-webapp` 專案，包含 `plugin.xml`、`WEB-INF/web.xml` 與靜態資源。Created the minimal `hello-webapp` project with `plugin.xml`, `WEB-INF/web.xml`, and static assets.
  • 把專案打包成 OSGi bundle，並產生 p2 update site。Packaged the project as an OSGi bundle and generated a p2 update site.
  • 將 update site 匯入 `updatesite.nsf`，再透過 `OSGI_HTTP_DYNAMIC_BUNDLES` 啟用。Imported the update site into `updatesite.nsf` and enabled it with `OSGI_HTTP_DYNAMIC_BUNDLES`.

這一步證明了什麼What this proved

  • Domino 可以提供非 XPages 應用。Domino can serve a non-XPages application.
  • OSGi bundle + p2 update site + updatesite.nsf 這條路是可行的。The OSGi bundle + p2 update site + updatesite.nsf path works in practice.
  • 之後的每一步都可以沿用相同部署機制，只替換前端內容。Later steps can keep the same deployment mechanism and only replace the front-end content.
