Vite 前端整合Vite Frontend Integration

步驟二的重點不是換掉 Domino，而是把前端改成正式的 build pipeline，讓 UI 有獨立的 source 與編譯流程。 Step 2 did not replace Domino. It replaced the handwritten front end with a real build pipeline and a separate source project.

做了哪些事What changed

  • 新增 `frontend-vite` 專案，作為前端 source 與 build 工作區。Added the `frontend-vite` project as the front-end source and build workspace.
  • 設定 Vite 輸出到 `WebContent/step2`。Configured Vite to emit its output into `WebContent/step2`.
  • 保留 `plugin.xml`、`WEB-INF/web.xml` 與 Domino 的部署流程不變。Kept `plugin.xml`, `WEB-INF/web.xml`, and the Domino deployment pipeline unchanged.

為什麼這樣切分Why this split works

  • 前端開發與 Domino 打包各自獨立，責任更清楚。Front-end development and Domino packaging stay separated cleanly.
  • 之後換框架或調整 UI，不必重設 Domino 架構。Future UI or framework changes do not require redesigning the Domino side.
  • 這一步讓教學站開始有「可維護前端」的基礎。This step establishes a maintainable front-end foundation for the tutorial site.
